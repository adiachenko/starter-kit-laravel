# Agent Rules Precedence

## Documentation Boundary

- Keep project-specific agent guidance in `.ai/guidelines/**` — do not modify `AGENTS.md` or `CLAUDE.md`.
- Keep `README.md` user/operator-facing; no agent-only implementation details.

## Instruction Precedence

`AGENTS.md` and `CLAUDE.md` are generated by Boost and combine multiple sources. Each section is marked with a header like `=== .ai/... rules ===` (project) or `=== ... rules ===` (Boost built-in).

When instructions conflict, resolve by priority:

1. **Project rules** (`=== .ai/` sections) — authoritative for this repository.
2. **Global user rules** — apply unless overridden by project rules.
3. **Boost built-in rules** — defaults; follow only when they don't conflict with (1) or (2).

Override only the conflicting part of a lower-priority rule; follow the rest as written.
